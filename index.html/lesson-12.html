SEMANTIC ELEMENTS

Semantic elements are tags that carry meaning about the structure and content of a webpage, making the HTML code more descriptive and meaningful.

These elements provide information about the type of content they contain, helping both browsers and developers understand the purpose of different sections in a document.

SEMANTIC ELEMENTs And DESCRIPTION
SEMANTIC ELEMENTs DESCRIPTION
<header>..</header> It is typically used to define introductory content at the top of a document or section.It may contain some heading elements but also a logo, a search form, an author name, and other elements.
<nav>..</nav> It represents a section of page that links to other pages or to parts the page: It is used to define a set of navigation links, It allows the creation of navigation bar or menu on the webpage.
<section>..</section> It is used to define a section in a document and often employed to group related content together, making it sematically meaningful for both browsers and developers.
<article>..</article> It is used to define a self-contained piece of content that could be distributed and reused independently. such as news article, blog post, or forum post.
<aside>..</aside> It is used to define content that is tangetially related to the content around it, often presented as a advertisement, or other supplementary content.
<details>..</details> It is used to create a disclosure widget from which the user can reveal additional information or controls. The visible content within the <details> tag serves as a summary or the content that is initially shown, and additional details are hidden until the user interacts with the disclosure widget.
<summary>..</summary> The <summary> element is used in conjuction with the <details> tag. It is represented by the <summary> tag and provides a visible heading or summary for the content within <details> element. The summary is what users see before interacting with the disclosure widget to reveal or hide additional details.
<main>..</main> It is used to define the main content of a document. It represents the central content area and should not be include content that is tangentially or supplementary to the main content. The <main> tag is typically employed to improve document structure accessibility.
<time>..</time> It is used to represent a specific period in time or a range of the time. It can includes date, time, or both. The datetime attribute is often used to provide machine-readable information about the time value.
<footer>..</footer> It is used to define the footer of a document or a section. It typically contains copyright information, links to related documents, or any other content that is considered as the footer of the page or section.
